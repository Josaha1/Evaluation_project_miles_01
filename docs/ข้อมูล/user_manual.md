## 1. ภาพรวมระบบ

### ระบบประเมิน 360 องศา กนอ. คืออะไร?

เป็นระบบประเมินผลบุคลากรแบบ 360 องศา พัฒนาด้วย **Laravel 11 + React (Inertia.js)** สำหรับการนิคมอุตสาหกรรมแห่งประเทศไทย (กนอ.) โดยรองรับการประเมินจากหลายมุมมอง:

| มุมการประเมิน | ภาษาอังกฤษ | คำอธิบาย |
|---|---|---|
| **บนลงล่าง** | Top-down | ผู้บังคับบัญชาประเมินผู้ใต้บังคับบัญชา |
| **ล่างขึ้นบน** | Bottom-up | ผู้ใต้บังคับบัญชาประเมินผู้บังคับบัญชา |
| **ระดับเดียวกัน** | Left (Peer) | เพื่อนร่วมงานระดับเดียวกันประเมินกัน |
| **ข้ามสายงาน** | Right (Cross) | ประเมินข้ามหน่วยงาน / ผู้ประเมินภายนอก |
| **ประเมินตนเอง** | Self | พนักงานประเมินตนเอง |

### กลุ่มผู้ถูกประเมิน

| กลุ่ม | ระดับ (Grade) | คำอธิบาย |
|---|---|---|
| ผู้ว่าการ | 13 | ผู้บริหารสูงสุด |
| ผู้บริหาร | 9–12 | ผู้บริหารระดับสูง–กลาง |
| พนักงาน | 4–8 | พนักงานทั่วไป |

### ปีงบประมาณ (Fiscal Year)

ระบบใช้ปีงบประมาณไทย: **ตุลาคม – กันยายน**
- เดือน 10–12 = ปีงบถัดไป (เช่น ต.ค. 2568 = ปีงบ 2569)
- แสดงเป็น พ.ศ. ในหน้า UI

## 3. บทบาทผู้ใช้งาน

### 3.1 ผู้ใช้ทั่วไป (role: user)

- เข้าสู่ระบบด้วยรหัสพนักงาน 6 หลัก
- ประเมินตนเอง (Self-evaluation)
- ประเมินผู้อื่นตามที่ได้รับมอบหมาย
- ดูสถานะการประเมินของตนเอง
- ทำแบบสำรวจความพึงพอใจ
- แก้ไขโปรไฟล์ส่วนตัว

### 3.2 ผู้ดูแลระบบ (role: admin)

- ทุกอย่างที่ User ทำได้ + :
- จัดการผู้ใช้งาน (เพิ่ม/แก้ไข/ลบ)
- จัดการโครงสร้างองค์กร (สายงาน/หน่วยงาน/ตำแหน่ง/ฝ่าย)
- สร้างและจัดการแบบประเมิน
- มอบหมายการประเมิน
- จัดการองค์กรภายนอกและ Access Code
- ดูรายงานผลการประเมิน
- Export ข้อมูล (Excel/PDF)

### 3.3 ผู้ประเมินภายนอก (External)

- เข้าระบบด้วย Access Code
- ยืนยันตัวตนและองค์กร
- ทำแบบประเมินตามที่กำหนด
- ประเมินได้ครั้งเดียวต่อ 1 Access Code

## 4. หน้าเว็บสำหรับผู้ใช้ทั่วไป (User)

### 4.1 หน้าแรก (Welcome Page)
**URL:** `/`

- หน้าแนะนำระบบ
- ปุ่มเข้าสู่ระบบ
- ข้อมูลทั่วไปเกี่ยวกับการประเมิน 360 องศา

### 4.2 เข้าสู่ระบบ (Login)
**URL:** `/login`

- กรอกรหัสพนักงาน (6 หลัก) + รหัสผ่าน
- มีประกาศกำหนดการประเมิน (Announcement Modal)
- มีตัวเลือก "จดจำฉัน" (Remember Me)
- ป้องกัน Brute-force: ล็อกหลังพยายาม 5 ครั้ง/นาที

### 4.3 แดชบอร์ดผู้ใช้ (User Dashboard)
**URL:** `/dashboard`

แสดงข้อมูล:
- **สถานะการประเมินตนเอง** — เสร็จแล้ว / ยังไม่เสร็จ
- **รายชื่อผู้ที่ต้องประเมิน** — พร้อมมุมการประเมิน (top/bottom/left/right)
- **ความคืบหน้า** — Progress bar ของแต่ละการมอบหมาย
- **แบบสำรวจความพึงพอใจ** — ลิงก์ทำแบบสำรวจ

### 4.4 ประเมินตนเอง (Self-Evaluation)
**URL:** `/evaluations/self`

**ขั้นตอนการทำ:**
1. เข้าหน้าประเมินตนเอง → ระบบโหลดแบบประเมินที่ตรงกับระดับ (grade) ของผู้ใช้
2. ตอบคำถามทีละขั้น (Step-by-step) แยกตามส่วน (Part)
3. แต่ละขั้นมีคำถามหลายข้อ — ตอบแบบ Rating (1–5), ข้อความ, หรือเลือกตอบ
4. ระบบบันทึกคำตอบอัตโนมัติ (Auto-save) ทุกข้อ
5. สามารถกลับมาทำต่อได้ (Resume)
6. กดส่งเมื่อตอบครบทุกข้อ

**ประเภทคำถาม:**
| ประเภท | คำอธิบาย |
|---|---|
| Rating | เลือกคะแนน 1–5 (น้อยที่สุด–มากที่สุด) |
| Open Text | ตอบข้อความอิสระ |
| Choice | เลือกคำตอบ 1 ข้อ |
| Multiple Choice | เลือกได้หลายข้อ |

### 4.5 ประเมินผู้อื่น (Assigned Evaluation)
**URL:** `/assigned-evaluations/{evaluateeId}`

**ขั้นตอน:**
1. เลือกผู้ถูกประเมินจากรายการในแดชบอร์ด
2. ดูข้อมูลผู้ถูกประเมิน (ชื่อ, ตำแหน่ง, หน่วยงาน)
3. ตอบคำถามทีละขั้น (Step-by-step)
4. มี **Peer Comparison Widget** — เปรียบเทียบคะแนนกับผู้ประเมินคนอื่นในมุมเดียวกัน
5. มี **Evaluatee Selector** — สลับไปประเมินคนอื่นได้
6. กดส่งเมื่อตอบครบ

### 4.6 แบบสำรวจความพึงพอใจ (Satisfaction Evaluation)
**URL:** `/satisfaction-evaluation/{evaluationId}`

แบบสำรวจ 8 ข้อ (คะแนน 1–5):
1. ความพึงพอใจต่อการใช้งานระบบประเมิน
2. ความง่ายในการใช้งาน
3. ความเร็วในการตอบสนองของระบบ
4. ความถูกต้องของข้อมูล
5. ความสะดวกในการเข้าถึง
6. ความครบถ้วนของข้อมูล
7. ความเหมาะสมของเนื้อหา
8. ความพึงพอใจโดยรวม

+ ช่องความคิดเห็นเพิ่มเติม (ข้อความอิสระ)

### 4.7 โปรไฟล์ (Profile)
**URL:** `/profile` | `/profile/edit`

- ดูข้อมูลส่วนตัว (ชื่อ, สายงาน, หน่วยงาน, ตำแหน่ง, ฝ่าย)
- แก้ไขข้อมูลส่วนตัว
- อัปโหลดรูปโปรไฟล์
- เปลี่ยนรหัสผ่าน
- สร้างหน่วยงาน/ฝ่าย/ตำแหน่งใหม่ได้ (Inline Create)

### 5.4 จัดการแบบประเมิน (Evaluation Management)

#### รายการแบบประเมิน
**URL:** `/evaluations`

| การดำเนินการ | คำอธิบาย |
|---|---|
| สร้างแบบประเมิน | กำหนดชื่อ, คำอธิบาย, ประเภทผู้ใช้, ช่วงระดับ |
| แก้ไข | แก้ไขข้อมูลทั่วไปของแบบประเมิน |
| ดูตัวอย่าง (Preview) | ดูแบบประเมินก่อน publish |
| เผยแพร่ (Publish) | เปลี่ยนสถานะเป็น active |
| ลบ | ลบแบบประเมินทั้งหมด (cascade) |

**สถานะแบบประเมิน:**
- `draft` — ร่าง, ยังแก้ไขได้
- `active` / `published` — ใช้งานจริง
- `inactive` — ปิดใช้งาน

#### จัดการส่วน (Parts)
**URL:** `/evaluations/{id}/parts`
- แบ่งแบบประเมินออกเป็นส่วนย่อย
- เรียงลำดับด้วย `order`

#### จัดการด้าน (Aspects)
**URL:** `/evaluations/{id}/parts/{partId}/aspects`
- เกณฑ์การประเมินในแต่ละส่วน
- กำหนดว่ามีด้านย่อย (Sub-aspects) หรือไม่

#### จัดการด้านย่อย (Sub-Aspects)
**URL:** `/evaluations/{id}/parts/{partId}/aspects/{aspectId}/subaspects`
- เกณฑ์ย่อยภายใต้ด้านหลัก (ใช้เมื่อจำเป็น)

#### จัดการคำถาม (Questions)
**URL:** `/evaluations/{id}/parts/{partId}/questions`
- สร้างคำถามภายใต้ส่วน/ด้าน/ด้านย่อย
- กำหนดประเภท: rating / open_text / choice / multiple_choice
- สร้างตัวเลือก (Options) พร้อมคะแนนสำหรับ rating

**โครงสร้างแบบประเมิน:**
```
แบบประเมิน (Evaluation)
├── ส่วนที่ 1 (Part)
│   ├── ด้านที่ 1 (Aspect)
│   │   ├── คำถาม 1 (Rating: 1-5)
│   │   └── คำถาม 2 (Open Text)
│   └── ด้านที่ 2 (Aspect, มี Sub-aspects)
│       ├── ด้านย่อย 2.1 (Sub-aspect)
│       │   └── คำถาม 3 (Choice)
│       └── ด้านย่อย 2.2
│           └── คำถาม 4 (Multiple Choice)
└── ส่วนที่ 2 (Part)
    └── ...
```

### 5.5 มอบหมายการประเมิน (Evaluation Assignment)
**URL:** `/admin/assignments`

| การดำเนินการ | คำอธิบาย |
|---|---|
| ดูรายการ | รายชื่อการมอบหมายทั้งหมด, กรองตามปีงบ |
| สร้างรายบุคคล | เลือกผู้ประเมิน → ผู้ถูกประเมิน → มุมประเมิน |
| สร้างจำนวนมาก (Bulk) | อัปโหลดหรือสร้างหลายรายการพร้อมกัน |
| แก้ไข | เปลี่ยนผู้ประเมิน / มุมประเมิน |
| ลบ | ลบรายบุคคล หรือ Bulk Delete |
| Export | ส่งออกข้อมูลการมอบหมาย |
