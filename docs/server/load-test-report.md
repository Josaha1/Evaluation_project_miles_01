## 3. วิธีการทดสอบ

### Scenario 1: Capacity Test (หา Threshold)
ค่อยๆ เพิ่มจำนวนผู้ใช้พร้อมกันจาก 10 → 25 → 50 → 75 → 100 → 150 → 200 → 300 → 500 คน โดยจำลองการเข้าหน้าเว็บ (หน้าแรก, หน้าเข้าสู่ระบบ, หน้าผู้ประเมินภายนอก) พร้อมจับเวลาตอบสนองและอัตรา error

### Scenario 2: Login Load Test (1000 คนพร้อมกัน)
จำลองผู้ใช้ 1,000 คนพร้อมกัน ทำการเข้าสู่ระบบ (GET login page → POST login) เพื่อทดสอบขีดจำกัดสูงสุด

## 5. สาเหตุของข้อจำกัด

ปัญหาหลัก **ไม่ใช่ Performance ของ Application** (ซึ่งตอบสนองเร็วมากที่ 64ms) แต่เป็น:

1. **Hostinger WAF/DDoS Protection** — ระบบป้องกันของ Hostinger ตัด TLS connection เมื่อมี concurrent connections สูงเกินกำหนด (~150-200 connections)
2. **Shared Infrastructure** — Cloud Enterprise ใช้ทรัพยากรร่วมกับลูกค้าอื่น ไม่สามารถ customize ได้
3. **ไม่มี Root Access** — ไม่สามารถปรับแต่ง PHP-FPM workers, Nginx config, หรือเพิ่ม Redis cache ได้

## 7. ข้อเสนอแนะ: ย้ายไป VPS KVM4 ของ Hostinger

### 7.1 เปรียบเทียบ Spec และราคา

| | **Cloud Enterprise (ปัจจุบัน)** | **VPS KVM4 (แนะนำ)** | **VPS KVM8** |
|---|---|---|---|
| **CPU** | 6 Cores (Shared) | 4 Cores (Dedicated) | 8 Cores (Dedicated) |
| **RAM** | 16 GB | 16 GB | 32 GB |
| **Storage** | 200 GB NVMe | 200 GB NVMe | 400 GB NVMe |
| **Bandwidth** | ไม่จำกัด* | 16 TB/เดือน | 32 TB/เดือน |
| **WAF Rate Limit** | มี (จำกัด concurrent) | ไม่มี (ควบคุมเอง) | ไม่มี (ควบคุมเอง) |
| **Root Access** | ไม่มี | มี | มี |
| **Custom Config** | ไม่ได้ | ได้เต็มที่ | ได้เต็มที่ |
| **ราคา/เดือน** | ~1,050 บาท | ~1,085 บาท | ~1,890 บาท |
| **ราคา promo ปีแรก** | - | ~350 บาท/เดือน | ~1,400 บาท/เดือน |

> *หมายเหตุ: อัตราแลกเปลี่ยนโดยประมาณ 1 USD = 35 THB*

### 7.2 ทำไมต้อง VPS KVM4

| ข้อ | รายละเอียด |
|-----|-----------|
| **1. ไม่มี WAF Rate Limit** | สามารถรับ 600+ concurrent connections ได้จริง ไม่ถูกบล็อก |
| **2. Root Access** | ปรับแต่ง PHP-FPM (เพิ่ม workers), Nginx (tuning), OPcache, Redis ได้เต็มที่ |
| **3. ราคาใกล้เคียงกัน** | ~$31/เดือน vs ~$30/เดือน (ต่างกันแค่ ~$1) |
| **4. Dedicated Resources** | CPU ไม่ต้องแชร์กับลูกค้าอื่น ทำให้ performance คงที่ |
| **5. Scalable** | อัพเกรดเป็น KVM8 ได้ง่ายหากต้องการในอนาคต |

### 7.3 ประมาณการรับ Concurrent Users หลังย้าย

| การปรับแต่ง | Concurrent Users ที่คาดว่ารับได้ |
|---|---|
| VPS KVM4 + Default Config | ~300-400 คน |
| VPS KVM4 + PHP-FPM Tuning + OPcache | ~500-700 คน |
| VPS KVM4 + Redis Session/Cache | ~700-1,000 คน |
| VPS KVM8 + Full Optimization | ~1,500+ คน |
